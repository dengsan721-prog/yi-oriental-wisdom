# 「艺」东方人生智慧产品设计规格

日期：2026-07-17

本规格是当前唯一产品基线。本轮补充重点为：页面级跳转、极简出生录入、付费级专业详批、全网内容来源治理，以及 GitHub Pages 与完整产品同源发布。

## 1. 产品目标

「艺」面向普通大众，不以命理专业人士为主要受众。产品以子平八字为专业主轴，通过四柱、十神、格局、调候、喜忌、大运与流年建立主结论；相面、面痣、手纹和星座作为独立传统观察，生肖、历史人物镜像与自然动物原型作为文化镜像，不与八字主结论混算。

产品要解决传统命理最核心的交付问题：术语很多，却没有把一个人讲明白。产品核心特色不是堆积术语，也不是将命理简化为一句标签，而是稳定输出三层解读：

1. 专业结论：保留传统命理叫法与判断依据。
2. 通俗翻译：用普通人能够立即理解的语言解释。
3. 场景与行动：说明结论在现实生活中可能怎样出现，以及可以采取什么行动。

界面以易用、紧凑、专业和顶级质感为准则。任何低频操作不得长期占据大面积空间；任何结果模块不得用大卡片承载少量文字。

产品对用户的核心承诺是：用专业命理建立判断，用生活故事把人讲明白，用东方镜像帮助记忆，用阶段建议帮助行动。

## 1.1 产品核心价值

1. 专业命理：算得有规则、有依据、可追溯。
2. 人生翻译：解释“我是什么样的人、为什么这样、现实中怎样出现”。
3. 东方镜像：通过历史人物、自然动物和传统技法增强理解、趣味与传播。
4. 人生陪伴：从固定命盘延伸到年度、月度、人生事件和行动复盘。
5. 关系智慧：帮助用户理解伴侣、子女、父母、朋友与商业伙伴。

## 1.2 产品长期目标

产品不能停留在一次性排盘。首次使用交付完整人生画像，后续通过年度地图、月度节奏、关系档案、人生事件与行动复盘形成持续使用价值。产品界面仅呈现产品能力，不展示价格、购买方案或销售文案。

## 2. 专家委员会质询结论

### 产品视角

问：用户第一次进入后最需要完成什么？

答：只需要快速完成出生信息，立即获得命盘。品牌介绍、体系说明、功能说明不得阻断主流程。

问：年份为何不能铺满页面？

答：年份是一次性输入，页面常驻展示会形成无效占位。年月日统一进入底部滚轮选择器。

### 交互视角

问：怎样区分阳历和农历，又不增加脑力负担？

答：在日期选择器顶部提供“阳历｜农历”二选一。默认阳历。农历模式支持月份、日期、闰月状态，并在确认区同步显示转换后的阳历日期供核对。

问：不知道时辰怎么办？

答：“不知道时辰”是正式选项，不是异常状态。选择后取消小时与分钟要求，排盘结果标注“时柱未定”，隐藏强依赖时柱的判断，并说明受影响范围。

### 命理视角

问：怎样避免结论粗糙？

答：所有命理解读必须能够追溯到命盘要素。每项解读包含专业结论、命盘依据、白话解释、典型场景、东方镜像、趋吉建议和需要留意七部分。不得使用固定文案冒充计算结果。

问：多种传统技法如何避免互相矛盾？

答：子平八字为主结论；相面、面痣、手纹和星座属于独立传统观察；生肖、历史人物与动物原型属于文化镜像。每种辅助技法也必须提供传统依据、白话解释、生活场景和行动建议，不得只展示一个标签。相面、面痣和手纹只提供标准参考照片与图谱供用户自行对照，产品不采集、不上传、不识别用户照片。

### 视觉视角

问：怎样同时实现恢弘质感与高信息密度？

答：使用深海蓝黑、温润金和克制的层次光效建立气势；使用紧凑数据带、排盘结构、标签、刻度和折叠层级承载信息。气势来自结构与节奏，不来自大面积空白。

### 普通用户视角

问：用户能否不懂命理也看明白？

答：默认展示“双层模式”，专业叫法与创新叫法同时出现，随后直接给白话场景。用户可切换为“通俗”或“专业”模式，但不要求用户预先理解术语。

## 3. 核心流程

`品牌启动 → 出生信息 → 排盘过程 → 东方人生画像 → 专业命理总览 → 命理详批 → 大运流年 → 东方镜像 → 关系合盘 → 传统技法 → 人生首页`

前三步必须是彼此独立的全屏页面状态，而不是同一长页面中的锚点：

- `#/`：品牌首页。
- `#/birth`：出生坐标录入页。
- `#/calculating`：排盘过程页。
- `#/report/<section>`：报告页及当前章节。

点击“开始排盘”前，页面 DOM 中不得呈现可通过上下滚动到达的录入区。点击后地址与页面同时切换；浏览器后退键返回上一页。刷新录入页时可恢复未提交草稿，刷新结果页时若会话内存在命盘则恢复当前章节，否则安全回到录入页。

启动页只保留：

- 单字品牌“艺”
- 两行品牌语：“看见命局”与“读懂时运”
- 主按钮“开始排盘”

首页删除所有小字说明、体系介绍、操作提示和功能列表。品牌字“艺”外圈采用缓慢、克制的呼吸光晕：缩放与透明度形成两层不同周期的起伏，不旋转、不闪烁；系统开启“减少动态效果”时停用呼吸，仅保留静态微光。

### 3.1 统一发布架构

完整 React 产品是唯一源码，GitHub Pages 只是同一产品的静态构建产物：

- 页面状态由轻量哈希路由管理，路由负责首页、录入、计算中、报告章节和人生首页之间的切换。
- 出生草稿、当前命盘和当前报告章节通过明确的会话状态接口传递，不写入 URL，也不依赖页面滚动位置。
- 命盘计算、详批、运势、镜像、合盘和传统技法继续使用 `site/lib/yi` 的同一套规则引擎。
- `docs/` 由构建命令生成，不再手工维护另一套内嵌 JavaScript 计算器。
- GitHub Pages 使用仓库相对基础路径加载资源，直接访问主页和刷新哈希路由均不能出现 404。

数据流为：

`出生草稿 → 历法转换与输入校验 → 四柱结构化命盘 → 专业规则判断 → 白话/场景/行动解释 → 分章节报告 → 会话内恢复或用户明确同意后的本机保存`

计算失败时保留出生草稿并返回录入页，不展示残缺报告；来源数据缺失时只降低对应模块置信度，不用默认文案冒充计算结果。

## 4. 出生信息录入

录入页顶部居中只写“建立出生坐标”，不展示“用最少操作完成录入”“请确认出生信息”或阴阳历说明。页面只展示六个必要信息：姓名、性别、历法、出生日期、出生时辰、出生地。姓名和出生地允许暂不填写；历法、日期与时辰为核心。

录入操作遵循“常用选项直接看到、低频解释按需打开”：

- 姓名与出生地使用单行输入，不展开说明段落。
- 性别、阳历/农历和时辰精度使用紧凑分段按钮。
- 日期与时间只显示当前选择和“修改”入口；滚轮只在弹层中出现。
- 双历转换结果只作为一行确认信息，不再占据独立大卡片。
- 真太阳时、未知时辰影响范围等必要边界放入信息按钮或结果边界卡，不在录入主流程堆字。

### 历法

- 使用“阳历｜农历”分段选择器。
- 默认阳历。
- 农历模式明确显示月份名称、日期名称和闰月状态。
- 选择完成后同时展示阳历与农历日期，供用户核对。
- 不要求用户理解历法转换规则。

### 日期滚轮

- 点击出生日期后，从屏幕底部弹出选择器。
- 年、月、日为三列上下滚动滚轮，选中行居中高亮。
- 支持滑动惯性、吸附定位、点击相邻项和键盘方向键。
- 年份范围 1900 年至当前年份。
- 日期随年份、月份和历法自动更新，不产生无效日期。
- 顶部显示历法切换，底部只有“取消”和“确认”。
- 选择器关闭后，录入页只显示一行已选日期，不保留年份网格。

### 时辰

- 提供“精确时间”“十二时辰”“不知道时辰”三种模式。
- 精确时间使用小时、分钟双列滚轮，并同步显示对应十二时辰。
- 十二时辰模式直接滚动选择子、丑、寅至亥时，并显示时间范围。
- 不知道时辰模式只需一次点击，不再显示其他时间控件。
- 未知时辰的结果置信度为“有限”，并逐项说明受影响内容。

### 出生地

- 字段名统一为“出生地址”，使用单行城市/区县输入，不在首屏展开地区列表。
- 当前版本仅记录地点；若尚未实现真太阳时校正，不得声称已经校正。
- 地址不完整时不阻断排盘；任何经纬度、时区或真太阳时校正都必须显示采用的地点与修正值，不能静默改变用户的出生时刻。

## 5. 排盘结果信息架构

结果页采用“大众人生画像 + 专业命理报告”的双层结构。顶部先用一句人格定位、核心特征、历史人物镜像和自然动物原型帮助用户迅速理解自己；紧接着展示专业命盘总览，证明结论有据可查。下面使用吸顶分区导航：

`画像｜命盘｜详批｜大运｜镜像｜合盘｜传统`

切换栏目时保留当前命盘，不重新计算。移动端使用横向可滑动吸顶导航；桌面端使用同一行导航。

### 东方人生画像

第一屏帮助普通用户快速回答“我是谁”，包括：

- 一句人格定位
- 三个核心特征
- “别人眼中的我”与“真实的我”
- 天赋使用说明
- 压力状态下的反应
- 当前人生主线任务
- 一个历史人物维度镜像
- 一个自然动物原型

画像必须由实际命盘结构生成，不得使用随机匹配。历史人物只比较特定能力、行为或人生节奏，不声称命运相同。动物原型通过自然行为翻译人格与压力模式，不使用贬损性动物标签。

### 专业命理总览

报告顶部先提供“出生与排盘事实带”，让普通用户确认这份报告确实属于自己：

- 公历出生日期与时间。
- 农历出生日期与时辰。
- 出生地址、采用时区及是否进行真太阳时校正。
- 生肖与太阳星座，其中星座明确标为现代文化辅助语言。
- 年柱、月柱、日柱、时柱与完整八字；未知时辰显示“时柱未定”。

专业命理总览在少量滚动范围内完成“先概括、再证明”，至少呈现：

- 四柱八字，未知时辰时显示“时柱未定”
- 日主、阴阳属性、月令与季节气候
- 月支本气、透干、各支藏干与根气
- 五行数量、分布、得令、得地、得势和结构支持度
- 缺失五行、较少五行与“是否真正需要”的分开说明
- 十神在天干与藏干中的完整分布及核心功能
- 天干五合、地支六合、三合、六冲、刑、害、破的命盘位置
- 格局观察、调候、喜用与忌神；引擎无法可靠判断时明确写“暂不作单一判断”
- 当前大运及起止年份
- 一段 120–220 字的精准总断
- 至少六条有命盘依据的关键判断
- 三条当下最重要的行动提示

总览使用紧凑数据块，不使用大面积空卡片。四柱为视觉核心，其余信息形成两到三列数据带。

五行“缺”只描述可见分布，不能直接推导喜用。产品必须区分：数量为零、数量偏少、失令、无根、受制、结构需要六种不同含义，避免“缺什么补什么”的常见误导。

### 命理详批

详批包含：性格命势、天赋能力、事业方向、财富结构、婚恋关系、子女家庭、健康节律七个主题。每个主题默认显示一段核心判断，点击后展开完整七层内容：

1. 专业判断：例如“食神透出、印星有根”。
2. 命盘依据：明确对应四柱、十神、旺衰、格局或合冲关系。
3. 白话翻译：例如“你擅长把复杂事情整理清楚，再讲给别人听”。
4. 典型场景：给出工作、关系或家庭中的具体表现。
5. 东方镜像：用人物经历或自然行为帮助理解，不替代专业判断。
6. 趋吉建议：给出可执行的选择、节奏或沟通动作。
7. 需要留意：使用克制语言描述倾向，不作灾祸断言。

每个主题至少包含三条由实际命盘要素生成的判断，不用同一套模板重复填充。

每条完整解读正文目标为 220–450 字；折叠前显示 40–90 字核心判断。七个主题合计不得少于二十一条结构化判断。相同专业依据可以在不同领域出现，但白话场景和行动建议不得重复。

### 大运流年

- 展示完整大运列表，每步包含干支、十神、起止年龄与起止年份。
- 当前大运突出显示，过去与未来大运弱化但可选择。
- 选择某步大运后显示：阶段主题、机会来源、主要挑战、事业、财富、关系、健康节律与行动建议。
- 在大运内部展示逐年流年列表；选择年份后显示流年与原局、大运之间的关系。
- 专业层标出相关天干地支、十神、合冲刑害；白话层说明现实中可能出现的场景。
- 不使用绝对吉凶、灾祸恐吓和确定性事件承诺。

每一步大运至少包含阶段气候、原局互动、机会来源、压力来源、事业、资源、关系、身心节律和行动策略九项；每一流年至少包含岁运关系、典型场景和年度动作三项。人生节奏不是年份列表，而是一段可看懂、可核对、可行动的阶段叙事。

### 生肖镜像

生肖模块不能只输出动物标签或固定性格词，至少包含：

1. 生肖与年支、五行和阴阳的基础关系。
2. 在民俗文化中的象征与历史来源。
3. 群体中的第一印象与建立信任方式。
4. 顺境中的优势策略。
5. 压力下可能过度使用的模式。
6. 工作、亲密关系和家庭三个具体场景。
7. 与日主、月令、十神主盘相同或相反之处。
8. 一条立即可做和一条长期训练建议。

生肖是年支文化镜像，不代表完整人格；其结论不得覆盖四柱主盘。

### 关系合盘

关系类型包括伴侣、子女、商业伙伴和朋友。先录入或选择对方命盘，再计算双方互动。结果包含：

- 五行互补与消耗
- 十神角色关系
- 天干地支合冲刑害
- 情绪与沟通节奏
- 决策和资源协作方式
- 容易发生分歧的现实场景
- 可执行的相处或合作规则

商业合盘必须增加权责、资金、节奏与退出机制提示；亲子合盘避免给孩子贴标签；伴侣合盘不使用“克夫”“克妻”等归罪表达。结果不输出单一“合或不合”分数。

### 传统技法

- 相面：按额、眉、眼、鼻、口、颌与整体比例提供标准参考照片、局部放大图和线稿标注；用户自行选择最接近的形态，再阅读传统相法术语、生活化翻译、场景、建议和与主盘的异同。
- 面痣：提供男女可切换的正面、侧面痣位图以及编号索引；用户自行点选位置，查看传统文化位置解读。模块不判断良恶性、疾病、寿命或健康风险，并提醒皮肤问题不属于本产品解读范围。
- 手纹：提供左右手掌参考照片、主要掌丘和主线图谱；用户自行对照手型、掌丘与清晰主线，左右手采用的口径必须明示，不推断疾病、遗传、寿命或身份。
- 星座：展示太阳星座的文化特征、表达方式、情感需要和工作模式，并与八字日主、月令和十神进行差异对照；不把占星符号称为天文学结论。

生肖、历史人物和自然动物原型归入“东方镜像”，不占用传统图像观察的四个入口。称骨从产品、规则库和来源列表中完全删除。

每一种传统技法都使用完整七层结构：

`专业结果 → 传统依据 → 通俗翻译 → 生活场景 → 优势与误区 → 行动建议 → 与主命盘对照`

传统技法提供三层阅读深度：第一层是一句话结果；第二层是依据、翻译和场景；第三层是出处、综合互证和深入建议。

### 传统图谱自查流程与边界

相面、面痣和手纹不是首次排盘的必填项，只在用户主动进入传统模块后开启：

`选择技法 → 选择观察部位 → 查看标准照片与图谱 → 用户自行对照 → 点选最接近形态 → 阅读传统解释 → 与主盘交叉参考`

- 产品页面不存在相机、上传、相册读取、人脸识别或掌纹识别入口。
- 用户的实际面部、痣位和手掌图像始终留在用户自己的设备和视线中，产品只记录其主动点选的图谱编号。
- 不提供颜值评分、种族民族推断、健康诊断、犯罪倾向或人格定罪。
- 每个参考特征至少提供两种照片角度或照片加线稿，避免用户因单张示例误判。
- 参考照片必须具有明确版权或使用授权；涉及可识别人物时必须有合适的肖像授权。优先使用专门拍摄的中性参考照片与原创标注图。
- 任何点选结果都允许随时修改或清除，且不改变八字主盘。

### 理论渊源

每个专业解读和传统技法底部显示一行简洁来源，例如：

> 理论依据：子平法·十神关系｜参考《滴天髓》《子平真诠》

点击“查看依据”后展开：

- 对应命盘数据
- 使用的判断规则
- 采用的流派口径
- 理论典籍或传统体系
- 产品的现代翻译说明
- 适用范围与参考边界

整份报告底部统一展示理论渊源与解读原则。理论出处与现代解释明确分开；没有明确依据的内容不得包装成传统定论；不同流派存在分歧时明确采用的口径。

## 5.3 全网内容研究与来源治理

全网搜索用于扩大资料面、寻找原典和补充生活化场景，不直接把搜索排名、营销文章或短视频口径写进产品。内容进入规则库前必须记录来源、版本、原句位置、采用口径、现代翻译、适用范围和复核状态。

来源按四级管理：

1. **A 级计算依据**：国家标准、天文历法机构资料、计算库官方文档与可复现测试。
2. **A/B 级传统原典**：可核对扫描本或完整文本的《三命通会》《滴天髓》《子平真诠》《渊海子平》《穷通宝鉴》等；不同版本与注本分开记录。
3. **B 级文化与人物资料**：国家博物馆、专业博物馆、正史、人物文集、学术论文和可靠传记。
4. **C 级线索资料**：大众命理网站、论坛、短视频和商业内容，只能用于发现问题、生活语言和竞品表达，不能单独支撑专业判断。

首批来源地图：

- 农历编算采用国家标准全文公开系统的 [GB/T 33661-2017《农历的编算和颁行》](https://openstd.samr.gov.cn/bzgk/std/newGbInfo?hcno=E107EA4DE9725EDF819F33C60A44B296) 与中国科学院紫金山天文台日历资料作校验基线。
- 命理与术数核心参考书目包括《渊海子平》《滴天髓》《子平真诠》《穷通宝鉴》《三命通会》《麻衣神相》《周易》《梅花易数》《神峰通考》《命理约言》等。优先使用可核扫描本、完整文本和学术整理本；识典古籍的 [《三命通会》总目与全文](https://www.shidianguji.com/zh/book/HY1521/chapter/1knwelu5suaf3)、[《滴天髓辑要》](https://www.shidianguji.com/book/NMG41601JH000040/chapter/1lly6pm5s2joo) 作为首批数字化入口。古籍原句、后世注解和产品解释必须三栏分开。
- 书目按用途分组：子平命理主轴使用《渊海子平》《滴天髓》《子平真诠》《穷通宝鉴》《三命通会》《神峰通考》《命理约言》；象数与变化语言参考《周易》《梅花易数》，但不得将卦象规则混入四柱计算；相面、面痣和手纹优先从《麻衣神相》及可核相学文献建立独立规则库。
- 生肖文化优先使用 [中国国家博物馆关于十二生肖的研究文章](https://www.chnmuseum.cn/yj/xscg/xslw/201812/t20181224_33168.shtml) 与博物馆藏品资料，不把生肖民俗直接当作八字主结论。
- 星座仅作为年轻用户熟悉的现代文化镜像，并明确区分占星符号与天文学星座；科学边界参考 [NASA 对星座与占星的说明](https://spaceplace.nasa.gov/constellations/sp/)。
- 历史人物镜像只使用有来源的人生经历与单维能力比较；没有可靠生辰时不计算“同命”，只展示行为或人生节奏的可核验相似维度。
- 产品采用“只展示标准图谱、不采集用户图像”的方案，从产品机制上避免处理人脸与指纹等生物识别信息；若未来改变这一机制，必须重新进行独立产品与合规评审，不能沿用本规格直接上线。

内容抓取必须遵守网站访问规则和著作权边界。产品保存规则摘要、短引用位置和链接，不复制受版权保护的整篇现代注解。任何自动抓取内容都要经过重复检测、矛盾检测、恐吓语言检测和人工可复核状态检查后才能上线。

## 5.1 持续使用系统

### 东方人生首页

完成首次排盘后，再次进入产品默认展示个人首页：

- 当前人生阶段
- 本月核心提醒
- 当前大运与流年
- 最近需要留意的事业、关系和生活节奏
- 尚未阅读的深度解读
- 已建立的关系档案
- 最近记录的人生事件
- 下一步行动建议

首页提供方向与节奏，不制造每日吉凶焦虑。

### 年度人生导航

每年形成一份年度地图，包含年度主线、事业、财富、关系、身心节律、十二个月趋势、适合推进事项、适合调整事项、年度行动计划和年末回顾。

### 月度节奏

每月提供核心主题、事业重点、关系重点、情绪与精力状态、典型场景、一件建议主动做的事、一件建议避免的事和上月行动回顾。

### 人生事件与行动复盘

用户可记录职业、关系、学习、居住、项目和家庭等重要事件。事件进入大运流年时间线，用于帮助用户观察判断与现实的对应关系、识别自己的重复模式并修正后续建议。产品不得把事件记录包装为命理必然验证。

### 个人智慧档案

长期保存用户的核心人格、优势证据、压力反应、重要选择、关系模式、事业轨迹、财富行为、人物镜像、动物原型、年度关键词和行动复盘。用户使用时间越长，档案越完整，但用户始终可以导出或删除。

### 关系星图

用户可以保存伴侣、子女、父母、朋友、商业伙伴和团队成员的关系档案。每段关系包含互动结构、沟通说明书、冲突场景、关系优势、相处建议、重要事件和当前阶段提醒。邀请对方建立档案必须基于对方主动授权，不读取通讯录制造关系链。

## 5.2 分享与年轻化表达

可生成用户主动选择的分享卡：东方人格、动物原型、历史人物镜像、年度关键词、关系说明书、别人眼中的我与真实的我、隐藏天赋和十年人生气候。

分享卡保持东方高级质感，不使用廉价测试题视觉；不展示完整出生日期、出生地点、未授权关系数据或专业命盘隐私。年轻化来自语言、互动、镜像、收藏和分享，不来自卡通化或娱乐化结论。

## 6. 解读生成规范

每条解读采用统一数据结构：

- `professionalTitle`：专业叫法
- `innovationTitle`：创新叫法
- `basis`：对应的四柱、十神、格局、旺衰或合冲依据
- `plainLanguage`：一到两句白话翻译
- `scenario`：具体生活场景
- `action`：明确、低风险、可执行的建议
- `caution`：需要留意的倾向
- `confidence`：高、中、有限
- `sourceTradition`：理论体系或流派
- `sourceReferences`：简洁典籍或资料来源
- `affectedByUnknownHour`：是否依赖时柱

未知时辰时，任何依赖时柱的条目标记为有限或直接隐藏。内容生成器不得将随机数、输入年份取模或固定索引包装为专业计算结果。

历史人物镜像必须记录人物资料来源、生辰可靠等级、匹配维度和相似度依据。自然动物原型必须记录与命盘人格结构的映射规则。不同技法发生矛盾时，以子平八字主盘为主，并向用户说明这是不同观察角度而非强行统一。

## 6.1 内容质量系统

内容系统分为：

1. 专业规则层：计算命盘并产生结构化判断。
2. 解释层：将术语翻译成普通语言。
3. 场景层：按职业、年龄、关系状态匹配现实例子。
4. 镜像层：匹配历史人物、动物原型和传统技法。
5. 行动层：生成低风险、可执行的建议。
6. 审核层：检查矛盾、重复、恐吓、绝对化和不当承诺。
7. 溯源层：保存理论出处、资料来源和内容版本。

同一命理结构对学生、职场人、创业者、照顾家庭者和退休人群使用不同场景，但专业判断保持一致。

丰富不等于无限堆字。内容密度按“结论是否有新增信息”验收：每个段落必须至少新增一项命盘依据、一个现实场景或一个行动方法；仅换同义词的扩写视为无效内容。系统同时建立术语词典、场景库、行动库、来源库和禁用断语库，使内容丰富但不互相矛盾。

## 7. 视觉与排版

- 基础色继续使用深海蓝黑、温润金、雾白和低饱和青灰。
- 字体以系统无衬线承载正文，宋体风格用于品牌字、四柱和章节标题。
- 正文控制在舒适行长；专业术语、白话和行动建议通过字号、颜色和背景层级区分。
- 卡片高度随内容变化，不设置无意义的大最小高度。
- 默认信息密度高于当前版本，但每个屏幕只保留一个明确视觉焦点。
- 触控目标不小于 44px；正文不小于 14px；重要数据不只依赖颜色表达。
- 动效只用于滚轮吸附、导航切换、折叠展开和排盘进度，并尊重减少动态效果设置。
- 页面切换使用 180–260 毫秒的淡入与轻微纵深变化；不通过滚动模拟页面跳转。
- 报告顶部采用紧凑事实带，专业骨架采用表格、关系标签和分布条，生活化解读使用可展开正文，避免所有内容都做成同一种大卡片。

## 8. 状态、错误与边界

- 非法日期在选择阶段即不可出现。
- 农历转换失败时保留用户输入并给出明确重试提示。
- 未知时辰不会阻止排盘。
- 计算失败时不显示部分伪结果，提供返回修改入口。
- 页面保存当前输入与浏览位置，栏目切换不丢失状态。
- 结果页持续显示传统文化参考边界，不作为医疗、法律、投资或重大人生决策依据。
- 用户可以导出、更正和删除出生信息、关系档案、人生事件与生成内容。
- 不满十四周岁用户的亲子关系数据和其他敏感信息必须经过监护人授权后处理。
- 相面、面痣、手纹不得出现相机、上传、相册读取或用户图像分析功能；若未来增加，必须先重新完成产品、隐私、安全与未成年人保护评审。
- 使用生成式人工智能生成内容时，产品需要具备内容标识、纠错反馈、投诉处理与版本追踪能力。

## 9. 验收标准

- 首页品牌语明确分为两行。
- 首页不能通过滚动看到出生录入；点击后必须进入独立 `#/birth` 页面，浏览器返回键有效。
- 首页删除小字说明；“艺”外圈存在克制呼吸动效，并支持减少动态效果。
- 录入页顶部居中只显示“建立出生坐标”，不显示设计目标式文案或阴阳历说明。
- 录入页包含出生地址，但地址为空时不阻断排盘。
- 年月日只通过紧凑入口和三列滚轮选择，不再展示年份网格、月份网格或日期网格。
- 阳历与农历明确可切换，并显示双历核对结果。
- 精确时间、十二时辰和不知道时辰均可在两次点击以内进入。
- 未知时辰能够正常生成有限结果，且不展示依赖时柱的确定性结论。
- 结果页包含画像、命盘、详批、大运、镜像、合盘和传统七个可切换栏目。
- 首次结果先帮助用户理解人生画像，再紧接专业命盘证明判断依据。
- 报告顶部完整展示公历、农历、出生地址、生肖、星座、年月日时四柱和时辰置信度。
- 命盘模块展示日主、月令、透干、藏干、根气、十神分布和干支关键关系；“五行有缺”不直接等于喜用。
- 每个详批主题同时包含专业依据、白话、场景、建议与提醒。
- 七域详批合计至少二十一条结构化判断，默认摘要紧凑，展开内容达到付费报告阅读密度。
- 大运能够选择阶段，流年能够选择年份，选择后内容同步变化。
- 每步大运包含九项阶段解读，每个流年包含至少三项现实化解读。
- 生肖镜像包含来源、关系方式、顺逆境、三个生活场景、主盘互证与行动建议。
- 合盘四类关系均包含专业关系、现实场景和相处方法。
- 传统技法包含相面、面痣、手纹和星座，每项具有完整七层解读；称骨不显示、不计算、不出现在来源列表。
- 相面、面痣和手纹只展示有版权来源的标准照片与图谱，由用户自行对照；页面不存在用户照片上传或自动识别。
- 东方镜像继续包含生肖、历史人物和自然动物原型，并与传统图像观察分区展示。
- 每个专业模块底部提供简洁理论来源，完整出处按需展开。
- 再次进入产品时展示人生首页，而不是重新要求排盘。
- 年度地图、月度节奏、人生事件、个人档案和关系星图具有明确入口。
- 分享卡不泄露出生信息和关系隐私。
- 移动端无横向页面溢出，吸顶导航与底部滚轮操作顺畅。
- 页面不存在 Demo、演示版、测试版或固定样例冒充真实结果的文案。
- 自动化测试、生产构建、GitHub Pages 与正式公网地址全部通过验证。
- GitHub Pages 必须由完整 React 产品构建生成，不再维护功能缩水的第二套内嵌计算器。
